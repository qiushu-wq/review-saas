📖 API 文档

智评助手商家集成 API — 将智能差评回复能力无缝接入您的系统

🔐 认证方式

智评助手提供两种认证方式，分别适用于不同的使用场景。所有需要认证的接口必须在请求头中携带有效的凭证。

方式一：JWT Token（商家后台使用）

🛡️
Bearer Token
登录商家后台后自动获取，适用于 Dashboard 内部调用。
Authorization: Bearer <token>

Token 在登录时由服务端颁发，存储在 localStorage 中。 调用 API 时需在请求头中附带。Token 过期后需重新登录获取。

方式二：API Key（外部集成使用）

🔑
X-API-Key
面向第三方系统集成，在 Dashboard 中创建并管理。
X-API-Key: <your-api-key>

API Key 适用于您的自有系统、第三方平台或自动化脚本调用。 您可以在「商家后台 → 设置」页面中创建和管理 API Key。每个商家可创建多个 Key，支持随时吊销。

💡 提示： 在 Dashboard 中创建的 API Key 请妥善保管，离开页面后将无法再次查看原文。 如怀疑 Key 泄露，请及时吊销并重新创建。

📡 基础信息

属性 值
Base URL http://localhost:3002/api
数据格式 所有请求和响应均为 application/json
字符编码 UTF-8
认证失败 返回 HTTP 状态码 401

💡 提示： 生产环境 Base URL 将替换为您的部署域名。 请在所有请求的 Content-Type 请求头中设置为 application/json。

🤖 生成回复

根据差评内容智能生成回复建议，支持三步结构（情绪安抚 → 原因核实 → 补偿方案）。

POST /api/external/generate

请求参数（Request Body）

字段类型必填说明
reviewContent string 必填 差评原文，建议长度不超过 2000 字
storeName string 选填 店铺名称，用于生成个性化回复
productName string 选填 商品或服务名称

响应结构（Response Body）

字段类型说明
reply string 完整回复文本
steps.comfort string 情绪安抚段落
steps.verify string 原因核实段落
steps.compensate string 补偿方案段落
severity string 严重程度：light / moderate / severe
source string 来源：ai（AI 生成）或 template（模板匹配）

cURL 示例

Bash 
curl -X POST http://localhost:3002/api/external/generate \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key-here" \
  -d '{
    "reviewContent": "菜量太少，价格太贵，服务态度很差，不会再来了。",
    "storeName": "老王川菜馆",
    "productName": "麻辣烫"
  }'

JavaScript (fetch) 示例

JavaScript 
// 使用 API Key 认证
const response = await fetch('http://localhost:3002/api/external/generate', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'your-api-key-here'
  },
  body: JSON.stringify({
    reviewContent: '菜量太少，价格太贵，服务态度很差，不会再来了。',
    storeName: '老王川菜馆',
    productName: '麻辣烫'
  })
})

const data = await response.json()
console.log(data.reply)
// 输出结构：
// {
//   "reply": "完整的回复文本...",
//   "steps": {
//     "comfort": "情绪安抚内容...",
//     "verify": "原因核实内容...",
//     "compensate": "补偿方案内容..."
//   },
//   "severity": "severe",
//   "source": "ai"
// }

📋 查看历史

分页查询商家的差评回复历史记录。

GET /api/external/history

查询参数（Query Parameters）

字段类型必填说明
page number 选填 页码，默认值为 1

响应结构（Response Body）

字段类型说明
items array 历史记录列表
total number 总记录数
page number 当前页码
totalPages number 总页数

cURL 示例

Bash 
curl "http://localhost:3002/api/external/history?page=1" \
  -H "X-API-Key: your-api-key-here"

JavaScript (fetch) 示例

JavaScript 
const response = await fetch('http://localhost:3002/api/external/history?page=1', {
  headers: {
    'X-API-Key': 'your-api-key-here'
  }
})

const data = await response.json()
console.log(data)
// 输出结构：
// {
//   "items": [
//     {
//       "id": 1,
//       "review_content": "菜量太少...",
//       "generated_reply": "完整的回复...",
//       "severity": "severe",
//       "source": "ai",
//       "created_at": "2026-05-19 10:30:00"
//     }
//   ],
//   "total": 42,
//   "page": 1,
//   "totalPages": 5
// }

📦 套餐信息

查询当前商家的套餐状态、使用额度和可选套餐列表。此接口需要 JWT 认证。

GET /api/plan/info
🛡️
JWT 认证要求
Authorization: Bearer <token>

响应结构（Response Body）

字段类型说明
currentPlan string 当前套餐：free / pro / enterprise
usedThisMonth number 本月已使用次数
monthlyLimit number 每月额度上限
availablePlans array 可选套餐列表

cURL 示例

Bash 
curl http://localhost:3002/api/plan/info \
  -H "Authorization: Bearer your-jwt-token"

⚠️ 错误码

API 使用标准的 HTTP 状态码表示请求结果。发生错误时，响应体将包含 error 字段说明具体原因。

400
参数错误
请求参数缺失或格式不正确，请检查请求体
401
认证失败
Token 无效、过期或 API Key 错误
404
资源不存在
请求的资源或接口路径不存在
409
资源冲突
资源已存在或状态冲突（如重复注册）
429
额度不足
本月使用额度已用完，请升级套餐或等待下月重置
500
服务器错误
服务器内部异常，请稍后重试

错误响应格式

JSON 
{
  "error": "认证失败，请检查 API Key 或 Token"
}
